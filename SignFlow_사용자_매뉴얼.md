## 2. 대시보드

로그인 후 처음 보이는 화면입니다.

| 구역 | 내용 |
|------|------|
| 통계 카드 | 전체 콘텐츠 수, 온라인 장치 수, 활성 스케줄 수, 스토리지 사용량 |
| 재생 트렌드 | 최근 7일간 콘텐츠 재생 횟수 꺾은선 그래프 |

| 최근 업로드 | 최근 등록된 콘텐츠 목록 |

숫자 카드를 클릭하면 해당 관리 페이지로 바로 이동합니다.

## 3. 콘텐츠 관리

### 3-1. 콘텐츠 업로드

1. 좌측 메뉴 **콘텐츠** 클릭
2. 우상단 **업로드** 버튼 클릭
3. 파일 선택 또는 드래그 앤 드롭
4. 이름, 태그, 생애주기 설정 후 저장

**지원 형식:** 이미지(JPG·PNG·GIF·WebP), 동영상(MP4·WebM), 오디오(MP3), HTML, 문서(PDF)

### 3-2. 콘텐츠 생애주기 (엠바고 & 만료)

콘텐츠가 자동으로 게시되거나 만료되도록 일정을 설정할 수 있습니다.

| 설정 | 설명 |
|------|------|
| 시작일 (엠바고) | 이 날짜 이후부터 화면에 표시됨. 미설정 시 즉시 게시. |
| 만료일 (라이프스팬) | 이 날짜 이후 자동으로 만료되어 재생 중단. |
| 비활성화 | 수동으로 즉시 콘텐츠를 숨김 처리. |

**게시 상태 표시:**
- 초록 = 게시 중
- 파랑 = 예약됨 (엠바고 기간)
- 회색 = 만료됨
- 빨강 = 비활성화
- 주황 = 곧 만료 (7일 이내)

### 3-3. 검색 및 필터

- 이름으로 검색
- 유형별 필터 (이미지/동영상/오디오/HTML/문서)
- 게시 상태별 필터
- 그리드/리스트 뷰 전환

| 시계 | 실시간 시각 표시 |
| 날씨 | 현재 날씨 정보 |

| 단축키 | 기능 |

## 5. 플레이리스트

콘텐츠 여러 개를 순서대로 묶은 재생 목록입니다.

### 5-1. 플레이리스트 생성

1. 좌측 메뉴 **플레이리스트** 클릭
2. **새 플레이리스트** 버튼 클릭
3. 이름과 유형 선택 후 생성

### 5-2. 플레이리스트 유형

| 유형 | 설명 |
|------|------|
| 일반 (GENERAL) | 순서대로 반복 재생. 가장 기본적인 형태. |
| 태그 기반 | 장치 태그 조건에 따라 다른 콘텐츠 재생 (→ 8장 참고) |
| 동기화 | 여러 장치가 동시에 같은 타이밍으로 재생 |
| 비디오월 | 스크린 월 전용 재생 목록 |
| 광고 | 광고 콘텐츠 전용 |

### 5-3. 플레이리스트 편집

카드의 **편집** 버튼을 클릭하면 에디터가 열립니다.

- **콘텐츠 추가:** 오른쪽 라이브러리에서 콘텐츠를 클릭
- **순서 변경:** 항목을 드래그 앤 드롭
- **재생 시간 변경:** 각 항목의 초(s) 값을 직접 수정
- **전환 효과:** 항목 사이 트랜지션 선택 (없음/페이드/슬라이드/줌/블러)
- **항목 삭제:** 항목 우측 X 버튼 클릭
- 변경 완료 후 **저장** 버튼 클릭

## 6. 스케줄

플레이리스트(또는 레이아웃)를 **언제**, **어느 장치**에서 재생할지 지정합니다.

### 6-1. 스케줄 생성

1. 좌측 메뉴 **스케줄** 클릭
2. **스케줄 생성** 버튼 클릭
3. 아래 항목 설정 후 **생성**

| 항목 | 설명 |
|------|------|
| 스케줄 이름 | 구분하기 쉬운 이름 |
| 콘텐츠 소스 | 플레이리스트 또는 레이아웃(멀티존) 선택 |
| 시작일 / 종료일 | 스케줄이 유효한 날짜 범위 |
| 시작 시간 / 종료 시간 | 하루 중 재생할 시간대 (예: 09:00 ~ 18:00) |
| 반복 | 없음 / 매일 / 매주 / 매월 |
| 장치 선택 | 이 스케줄을 받을 장치 (Ctrl+클릭으로 다중 선택) |

### 6-2. 스케줄 배포

생성 후에는 상태가 **DRAFT(초안)**입니다. 장치에 실제 적용하려면 **배포**가 필요합니다.

1. 스케줄 카드 또는 캘린더 이벤트 클릭
2. 상세 모달에서 **배포** 버튼 클릭
3. 시간대 충돌이 있으면 경고 다이얼로그가 표시됨
   - **취소** = 배포 중단
   - **강제 배포** = 기존 스케줄과 겹쳐도 강제 적용

### 6-3. 스케줄 상태

| 상태 | 의미 |
|------|------|
| DRAFT | 배포 전 초안 상태 |
| ACTIVE | 장치에 배포되어 재생 중 |
| PAUSED | 일시 중지 |
| CANCELLED | 취소됨 |

### 6-4. 조건부 재생 (스케줄 내)

스케줄 상세 화면 하단의 **조건부 재생** 섹션에서, 장치 태그 조건에 따라 다른 플레이리스트를 재생하도록 규칙을 추가할 수 있습니다. 자세한 내용은 **8장** 참고.

## 7. 채널

채널은 **"기본 재생 목록"** 개념입니다. 스케줄이 없거나 스케줄 시간이 아닐 때 장치가 재생할 콘텐츠를 지정합니다.

### 7-1. 채널이란?

```
스케줄이 있으면  →  스케줄의 플레이리스트를 재생
스케줄이 없으면  →  채널의 콘텐츠를 재생 (항상 켜진 기본 화면)
```

예를 들어 매장 홍보 영상을 항상 틀어 놓고, 점심시간(12:00~13:00)에만 메뉴판 스케줄을 넣고 싶다면:
- **채널:** 홍보 영상
- **스케줄:** 메뉴판 플레이리스트, 12:00~13:00, 매일

### 7-2. 채널 생성

1. 좌측 메뉴 **채널** 클릭
2. **새 채널** 버튼 클릭
3. 이름과 설명 입력 후 생성

### 7-3. 채널에 콘텐츠 추가

1. 채널 카드의 **콘텐츠 관리** 버튼 클릭
2. 콘텐츠 라이브러리에서 추가할 콘텐츠 선택
3. **추가** 버튼 클릭

### 7-4. 채널에 장치 연결

채널은 **장치에 직접 연결**해야 합니다.

1. 채널 카드의 **장치 관리** 버튼 클릭
2. 연결할 장치 체크박스 선택
3. **저장** 버튼 클릭

한 장치는 하나의 채널에만 연결됩니다.

### 7-5. 기본 채널 설정

별표(★) 아이콘이 표시된 채널이 **기본 채널**입니다. 기본 채널은 특별한 예약이 없을 때 우선 재생됩니다.

채널 생성 시 **기본 채널로 설정** 옵션을 활성화하면 됩니다.

## 8. 태그 기반 조건부 재생

**장치마다 다른 콘텐츠**를 자동으로 재생하는 고급 기능입니다. 태그(Key:Value) 값에 따라 재생 규칙이 달라집니다.

### 8-1. 개념 이해

```
장치 A  →  태그: location=entrance   →  입구 홍보 영상 재생
장치 B  →  태그: location=checkout   →  계산대 프로모션 재생
장치 C  →  태그: floor=2            →  2층 전용 콘텐츠 재생
```

같은 스케줄을 여러 장치에 배포해도, 태그 조건에 따라 각 장치가 다른 플레이리스트를 재생합니다.

### 8-2. 장치에 태그 설정

1. 좌측 메뉴 **태그 재생** 클릭
2. 장치 목록에서 원하는 장치의 **태그 편집** 버튼 클릭
3. Key:Value 형식으로 태그 추가
   - 예: `location` = `entrance`
   - 예: `floor` = `2`
   - 예: `store` = `gangnam`
4. **저장** 클릭

### 8-3. 조건부 재생 규칙 설정

스케줄 상세 화면의 **조건부 재생** 섹션에서 규칙을 추가합니다.

1. 스케줄을 클릭하여 상세 모달 열기
2. **조건부 재생** 섹션에서 **규칙 추가** 클릭
3. 조건 설정:
   - **태그 키** 선택 (예: `location`)
   - **태그 값** 입력 (예: `entrance`)
   - **재생할 플레이리스트** 선택
4. 저장

**동작 방식:**
- 조건 규칙이 있으면 → 태그 조건이 일치하는 장치에만 해당 플레이리스트 재생
- 일치하는 규칙이 없으면 → 스케줄의 기본 플레이리스트 재생
- 스케줄 시간이 아니면 → 채널 콘텐츠 재생

### 8-4. 태그 현황 확인

**태그 재생** 페이지에서 전체 태그 현황을 한눈에 볼 수 있습니다.
- 사용 중인 태그 키/값 목록
- 태그별 장치 수
- 특정 태그로 필터링하여 해당 장치 확인

## 9. 장치 관리

### 9-1. 장치 등록

1. 좌측 메뉴 **장치** 클릭
2. **새 장치** 버튼 클릭
3. 아래 정보 입력

| 항목 | 설명 |
|------|------|
| 장치 이름 | 구분하기 쉬운 이름 (예: 1층 입구 모니터) |
| 장치 ID | 플레이어 앱의 고유 식별자 |
| IP 주소 | 장치의 내부 IP |
| MAC 주소 | 장치의 MAC 주소 |
| 모델 | 하드웨어 모델명 |
| 해상도 | 화면 해상도 |
| 위치 | 설치 위치 메모 |
| 매장 | 소속 매장 |
| 그룹 | 장치 그룹 |

### 9-2. 장치 상태

| 상태 | 의미 |
|------|------|
| 온라인 (초록) | 서버와 연결 중 |
| 오프라인 (회색) | 연결 끊김 |
| 경고 (주황) | 이상 감지 |

### 9-3. 장치 검색 및 필터

- 장치 이름/ID로 검색
- 상태별 필터 (온라인/오프라인/경고)
- 매장별 필터

## 10. 모니터링

등록된 모든 장치의 **실시간 화면**을 한 페이지에서 확인합니다.

### 10-1. 스크린샷 보기

- 각 장치 카드에 최근 캡처된 화면이 표시됩니다.
- 카드를 클릭하면 전체 화면으로 볼 수 있습니다.

### 10-2. 스크린샷 새로고침

| 방법 | 설명 |
|------|------|
| 전체 새로고침 버튼 | 모든 장치에 스크린샷 캡처 요청 |
| 장치별 새로고침 버튼 | 해당 장치만 즉시 캡처 |
| 자동 새로고침 | 30초 / 1분 / 2분 / 5분 간격으로 자동 갱신 |

캡처 시각이 카드 하단에 표시됩니다.

## 11. 스크린 월 & 동기화

### 11-1. 스크린 월 (멀티 모니터 벽면)

여러 대의 모니터를 격자로 배열하여 하나의 큰 화면처럼 사용하는 기능입니다.

**스크린 월 생성:**
1. 좌측 메뉴 **스크린 월** → **스크린 월** 탭
2. **새 스크린 월** 클릭
3. 이름, 행×열 구성, 베젤 보정값(mm) 설정
4. 격자에서 셀을 클릭하여 장치 배치

**베젤 보정:** 모니터 테두리(베젤) 두께만큼 콘텐츠를 조정하여 이음새가 자연스럽게 이어지도록 합니다.

### 11-2. 동기화 그룹

여러 장치가 **정확히 같은 타이밍**에 콘텐츠를 재생하도록 동기화합니다.

**동기화 그룹 생성:**
1. **스크린 월** → **동기화 그룹** 탭
2. **새 그룹** 클릭
3. 이름, 동기화 모드, 장치 선택, 마스터 장치 지정

**동기화 모드:**

| 모드 | 정밀도 | 사용 환경 |
|------|--------|-----------|
| LAN | ±50ms | 같은 네트워크 내 고정밀 동기화 |
| WAN | ±500ms | 인터넷 환경, 정밀도가 낮음 |

## 12. 긴급 방송

긴급 상황 시 모든 장치 또는 특정 장치에 즉시 메시지를 표시합니다.

> 이 기능은 **STORE_MANAGER 이상** 권한이 필요합니다.

### 12-1. 긴급 메시지 전송

1. 좌측 메뉴 **긴급 방송** 클릭
2. 아래 항목 설정 후 **전송**

| 항목 | 설명 |
|------|------|
| 제목 | 긴급 메시지 제목 |
| 내용 | 표시할 메시지 내용 |
| 유형 | 정보(파랑) / 경고(주황) / 위험(빨강) / 커스텀 |
| 표시 방식 | 오버레이(화면 위에 덧씌움) / 전체화면 / 티커(하단 자막) |
| 자동 종료 | 30분 / 1시간 / 3시간 / 24시간 후 자동 해제 |
| 대상 | 전체 장치 / 특정 매장 / 특정 장치 선택 |

### 12-2. 메시지 해제

**현재 활성 메시지** 목록에서 **비활성화** 버튼을 클릭하면 즉시 화면에서 사라집니다.

### 12-3. 미리보기

설정한 색상, 폰트, 유형으로 실제 화면에 어떻게 보일지 오른쪽 미리보기 영역에서 확인할 수 있습니다.

## 13. 레이아웃 (멀티존)

하나의 화면을 **여러 영역(존)**으로 나누어 서로 다른 콘텐츠를 동시에 표시하는 기능입니다.

```
┌─────────────────┬──────────┐
│                 │  날씨    │
│   메인 영상     ├──────────┤
│                 │  뉴스    │
└────────────────────────────┘
```

### 13-1. 레이아웃 생성

1. 좌측 메뉴 **레이아웃** 클릭
2. **새 레이아웃** 버튼 클릭
3. 에디터에서 영역(존)을 추가하고 각 존에 콘텐츠 할당
4. 기본 해상도 설정 (예: 1920×1080)

### 13-2. 레이아웃을 스케줄에 적용

스케줄 생성 시 **콘텐츠 소스** 탭에서 **레이아웃(멀티존)**을 선택하면 됩니다.

## 15. 콘텐츠가 화면에 표시되는 원리

복잡해 보이지만 우선순위가 명확합니다.

```
장치에 콘텐츠가 표시되는 우선순위

1순위: 긴급 방송 (활성화 중이면 모든 것을 가리고 표시)
        |
2순위: 스케줄 (현재 시간에 해당하는 활성 스케줄)
        ├─ 조건부 재생 규칙이 있으면 → 태그 조건에 맞는 플레이리스트
        └─ 없으면 → 스케줄에 지정된 기본 플레이리스트
        |
3순위: 채널 (스케줄이 없는 시간대의 기본 콘텐츠)
        |
4순위: 아무것도 없으면 → 빈 화면
```

### 실제 적용 예시

**매장 시나리오:**
- 영업시간 전체 (00:00~24:00): 채널에 브랜드 홍보 영상 설정
- 점심 프로모션 (11:00~14:00): 스케줄로 메뉴판 플레이리스트 배포
- 층별 다른 콘텐츠: 장치 태그 `floor=1`, `floor=2` 설정 후 조건부 재생 규칙 추가
- 화재 대피 안내: 긴급 방송으로 즉시 전체 화면 표시

## 자주 묻는 질문

**Q. 스케줄을 만들었는데 장치에 아무것도 안 보여요.**
→ 스케줄 상태가 DRAFT인지 확인하세요. **배포** 버튼을 눌러야 장치에 적용됩니다.

**Q. 채널을 만들었는데 장치에 재생이 안 돼요.**
→ 채널에 장치가 연결되어 있는지 확인하세요. 채널 카드의 **장치 관리** 버튼에서 장치를 추가해야 합니다.

**Q. 여러 장치에 같은 스케줄을 넣었는데 다른 내용이 재생되게 하고 싶어요.**
→ 8장 **태그 기반 조건부 재생**을 사용하세요. 장치에 태그를 설정하고 스케줄에 조건부 재생 규칙을 추가하면 됩니다.

**Q. 스크린샷이 업데이트가 안 돼요.**
→ 모니터링 페이지에서 **전체 새로고침** 버튼을 누르거나, 자동 새로고침 간격을 설정하세요.

**Q. 긴급 방송을 보냈는데 일부 장치에만 표시돼요.**
→ 긴급 방송 대상 설정을 확인하세요. 특정 매장/장치로 제한되어 있을 수 있습니다.
