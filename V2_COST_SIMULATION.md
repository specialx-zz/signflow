### 옵션 A: 직접 서버 스트리밍 (현재 방식)

> 모든 미디어를 본사 서버에서 실시간 전송

**트래픽 계산 (성장기 기준, 150대):**

```
시나리오: 디바이스당 하루 8시간 재생
  - 이미지: 평균 10초 표시, 시간당 360장 × 3MB = 1,080 MB/시간
    → 하지만 같은 이미지 반복이므로 캐시 적용 시 초회만 전송
  - 동영상: 30초짜리 100MB, 시간당 120회 = 이론상 12GB/시간
    → 같은 영상 반복이므로 실제로는 브라우저 캐시 히트

실제 전송량 (브라우저 캐시 미스 가정, 스케줄 변경 시):
  - 주 2회 스케줄 변경 × 디바이스당 평균 500MB 신규 콘텐츠
  - = 주 1GB/디바이스 × 150대 = 150 GB/주 = 600 GB/월

문제: 브라우저 캐시가 불안정 (탭 새로고침, 메모리 부족 시 캐시 삭제)
  - 예상 캐시 미스율: 20~40%
  - 추가 반복 전송: 월 200~400 GB 추가 가능
```

**서버 비용:**

| 항목 | 초기 | 성장기 | 안정기 |
|------|------|-------|-------|
| VPS/서버 (본사) | 5만원/월 | 10만원/월 | 30만원/월 |
| 트래픽 (1TB 초과분) | 0원 | 5~15만원/월 | 30~80만원/월 |
| 디스크 | 포함 | 5만원/월 | 15만원/월 |
| **월 합계** | **5만원** | **20~30만원** | **75~125만원** |

**문제점:**
- 대역폭 병목: 서버 1Gbps 회선에 500대가 동시 다운로드하면 대역폭 부족
- 안정성: 서버 다운 = 전 디바이스 콘텐츠 재생 불가
- 확장성: 디바이스 늘어날수록 트래픽 비용 급증
- 브라우저 캐시 불안정: 예측 불가능한 트래픽 스파이크

### 옵션 B: AWS S3 + CloudFront CDN (풀세트)

> 업로드 → S3 저장 → CloudFront CDN → 디바이스

**AWS 단가 (서울 리전, 2026 기준 추정):**

```
S3 Standard:
  - 저장:     $0.025/GB/월 (= 약 33원/GB/월)
  - PUT 요청: $0.005/1,000건
  - GET 요청: $0.0004/1,000건

CloudFront:
  - 첫 10TB:  $0.120/GB (= 약 160원/GB)
  - 다음 40TB: $0.100/GB
  - 50TB 이상: $0.080/GB

데이터 전송 (S3 → 인터넷, CDN 미사용 시):
  - 첫 10TB: $0.126/GB
  - CloudFront 통하면 S3→CF는 무료
```

**비용 계산:**

#### 초기 (5업체, 30대)

```
콘텐츠 전송 시나리오:
  - 주 2회 스케줄 변경 × 30대 × 500MB = 30GB/주 = 120GB/월
  - 한번 다운로드하면 로컬 캐시 → 반복 전송 없음!

S3 저장: 20GB × $0.025 = $0.50 (약 670원)
S3 요청: PUT 200건 + GET 3,000건 = 무시 가능
CloudFront 전송: 120GB × $0.120 = $14.40 (약 19,300원)
────────────────────────────────────────
월 합계: 약 $15 = 약 2만원
```

#### 성장기 (20업체, 150대)

```
콘텐츠 전송:
  - 주 2회 × 150대 × 500MB = 150GB/주 = 600GB/월

S3 저장: 200GB × $0.025 = $5.00 (약 6,700원)
CloudFront 전송: 600GB × $0.120 = $72.00 (약 96,500원)
────────────────────────────────────────
월 합계: 약 $77 = 약 10.3만원
```

#### 안정기 (50업체, 500대)

```
콘텐츠 전송:
  - 주 2회 × 500대 × 500MB = 500GB/주 = 2TB/월

S3 저장: 1TB × $0.025 = $25.60 (약 34,300원)
CloudFront 전송: 2TB × $0.120 = $245.76 (약 33만원)
────────────────────────────────────────
월 합계: 약 $271 = 약 36.3만원
```

**요약:**

| 시기 | S3 저장 | CDN 전송 | 합계 |
|------|--------|---------|------|
| 초기 (30대) | 670원 | 1.9만원 | **약 2만원/월** |
| 성장기 (150대) | 6,700원 | 9.7만원 | **약 10.3만원/월** |
| 안정기 (500대) | 3.4만원 | 33만원 | **약 36.3만원/월** |

### 옵션 C: S3 호환 스토리지 + 플레이어 로컬 다운로드 (CDN 없이) ⭐ 추천

## 5. 추천 전략: 단계적 접근

### Phase 1 (초기): 옵션 D — 자체 서버 + 로컬 다운로드

```
왜: 초기에는 클라우드 계정 설정/연동 없이 빠르게 시작
어떻게:
  - 현재 서버에 파일 저장 (기존 방식 유지)
  - 플레이어에 "다운로드 매니저" 추가 (핵심 개발)
  - 콘텐츠를 1회 다운로드 → 로컬 저장 → 로컬 재생
  - 서버 트래픽: 스트리밍에서 다운로드로 전환만으로 대폭 감소

비용: 기존 서버 비용만 (추가 비용 0원)
개발량: 플레이어 다운로드 매니저만 구현하면 됨
```

### Phase 2 (성장기): 옵션 C — Cloudflare R2 전환

```
왜: 디바이스 100대 이상이면 서버 대역폭 부담 시작
어떻게:
  - 파일 업로드 경로를 R2로 전환
  - 기존 파일 마이그레이션 스크립트
  - 플레이어 다운로드 URL만 R2 URL로 변경
  - 서버는 API + Socket.IO만 담당 (미디어 전송 안 함)

비용: 월 4,000원 수준
전환 난이도: 낮음 (S3 호환 API이므로 aws-sdk 그대로 사용)
```

### Phase 3 (대규모): 옵션 B 부분 적용 — CDN 추가

```
왜: 해외 진출 또는 디바이스 1000대 이상일 때
어떻게:
  - Cloudflare CDN은 R2와 자동 연동 (추가 설정만)
  - 또는 CloudFront 연동
  - 지역별 엣지 캐시로 다운로드 속도 향상

비용: Cloudflare 무료 플랜으로도 CDN 캐싱 가능!
  (R2 + Cloudflare CDN 조합이면 전송 무료 유지)
```

## 6. Cloudflare R2 + 로컬 다운로드 구현 상세

### 6.1 업로드 흐름

```
관리자                    백엔드 서버              Cloudflare R2
  │                         │                        │
  │  파일 업로드 (POST)      │                        │
  ├────────────────────────→│                        │
  │                         │  S3 PutObject           │
  │                         ├───────────────────────→│
  │                         │  ← URL 반환             │
  │                         │←───────────────────────┤
  │  ← 업로드 완료 + URL     │                        │
  │←────────────────────────┤                        │
```

### 6.2 다운로드 흐름 (플레이어)

```
플레이어                   백엔드 서버              Cloudflare R2
  │                         │                        │
  │  스케줄 수신 (Socket)    │                        │
  │←────────────────────────┤                        │
  │                         │                        │
  │  매니페스트 요청          │                        │
  ├────────────────────────→│                        │
  │  ← 콘텐츠 목록 + URL     │                        │
  │←────────────────────────┤                        │
  │                         │                        │
  │  콘텐츠 직접 다운로드 (서버 경유 안 함!)             │
  ├──────────────────────────────────────────────────→│
  │  ← 파일 바이너리                                   │
  │←──────────────────────────────────────────────────┤
  │                         │                        │
  │  다운로드 완료 보고       │                        │
  ├────────────────────────→│                        │
```

### 6.3 백엔드 코드 변경 (개념)

```javascript
// 현재: 로컬 파일 저장
app.post('/api/content/upload', upload.single('file'), async (req, res) => {
  // multer가 /uploads/에 저장
  const fileUrl = `/uploads/${req.file.filename}`
  // DB에 fileUrl 저장
})

// V2: R2에 저장 (S3 호환 SDK 사용)
const { S3Client, PutObjectCommand } = require('@aws-sdk/client-s3')
const s3 = new S3Client({
  region: 'auto',
  endpoint: 'https://<ACCOUNT_ID>.r2.cloudflarestorage.com',
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY,
    secretAccessKey: process.env.R2_SECRET_KEY,
  }
})

app.post('/api/content/upload', upload.single('file'), async (req, res) => {
  const key = `${req.tenantId}/${req.file.filename}`

  await s3.send(new PutObjectCommand({
    Bucket: 'signflow-content',
    Key: key,
    Body: fs.createReadStream(req.file.path),
    ContentType: req.file.mimetype,
  }))

  // 공개 URL 또는 Signed URL
  const fileUrl = `https://content.signflow.co.kr/${key}`
  // DB에 fileUrl 저장

  // 로컬 임시 파일 삭제
  fs.unlinkSync(req.file.path)
})
```
