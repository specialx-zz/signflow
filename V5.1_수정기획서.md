# VueSign V5.1 종합 수정 기획서

## 📋 개요

V5 시스템 전체 검수 후 발견된 문제점과 수정 내역을 정리한 문서입니다.
운영 흐름의 끊김(Flow 단절), UI/UX 미비, 백엔드 로직 오류를 체계적으로 수정합니다.

---

## ✅ 수정 완료 항목

### 1. 캔버스 에디터 — 위젯 드래그/이동 불가 (Critical)

**문제**: 위젯(시계, 날씨 등)을 캔버스에 추가한 후 마우스로 이동/크기 조절이 불가능
**원인**: 위젯 HTML 오버레이의 `pointerEvents: 'auto'`와 높은 `zIndex`가 Fabric.js 프록시 Rect의 마우스 이벤트를 차단
**수정**:
- `CanvasStage.tsx`: 위젯 오버레이에 `pointerEvents: 'none'` 적용
- Fabric 프록시 Rect에 더 명확한 선택 핸들(파란색 원형 코너, 점선 테두리) 추가
- 불필요한 `handleWidgetClick` 핸들러 제거

### 2. 매장-장치 배정 UI (Critical)

**문제**: 매장은 등록되어 있지만 장치를 매장에 배정하는 UI가 없음
**수정**:
- `StoresPage.tsx` 전면 개편:
  - **장치 관리**: 매장별 배정된 장치 목록 확장 패널 (펼침/접기)
  - **장치 배정**: 미배정 장치를 매장에 배정하는 모달 (배정 버튼)
  - **장치 해제**: 배정된 장치를 해제하는 기능 (해제 아이콘)
- `stores.ts` API: `getDevices()`, `assignDevice()`, `removeDevice()` 추가

### 3. 장치 등록코드 관리 UI (Critical)

**문제**: 백엔드에 토큰 API가 있지만 프론트엔드에서 관리 UI 없음
**수정**:
- `StoresPage.tsx`에 등록코드 관리 모달 추가:
  - 매장별 등록코드 생성 (이름, 만료시간 설정)
  - 생성된 코드 목록 (사용 여부, 만료일, 복사/삭제)
  - 등록코드 워크플로우 안내 메시지
- `devices.ts` API: `getTokens()`, `createToken()`, `deleteToken()` 추가

### 4. 장치 상세 편집 UI (Critical)

**문제**: 장치 등록 후 정보 수정 불가 (읽기 전용만 존재)
**수정**:
- `DeviceDetailPage.tsx` 전면 개편:
  - **편집 모드**: 장치 이름, IP, 모델, 해상도, 매장, 그룹, 위치 편집
  - 매장/그룹 드롭다운 선택
  - 저장/취소 버튼
  - 매장 배지 표시 (헤더에 매장명)
  - 매장 관련 데이터(`store` relation) 포함

### 5. STORE_MANAGER 매장 범위 제한 (High)

**문제**: STORE_MANAGER가 모든 장치를 볼 수 있음 (자기 매장만 봐야 함)
**수정**:
- `routes/devices.js`: `storeFilter` 미들웨어 추가
- `storeFilter`는 STORE_MANAGER의 `user.storeId`를 `req.storeId`에 설정
- `getDevices` 컨트롤러는 `req.storeId`로 자동 필터링

### 6. Device 타입에 store 관계 추가

**문제**: `Device` 인터페이스에 `storeId`와 `store` 관계가 누락
**수정**:
- `types/index.ts`: `storeId`, `store` 필드 추가
- `DeviceRegistrationToken` 인터페이스 신규 추가
- `DevicesPage.tsx`: 타입 캐스팅 제거, 정식 타입 사용

### 7. 백엔드 getDeviceById에 store 관계 포함

**문제**: 장치 상세 조회 시 `store` 정보가 빠져있음
**수정**: `deviceController.js`의 `getDeviceById`에 `store: { select: { id, name } }` include 추가

### 8. 캔버스 요소/위젯 추가 시 화면 중앙 배치 (V5.1.1)

**문제**: 요소나 위젯을 추가하면 x:100, y:100에 고정 → 화면이 작으면 안 보임
**수정**:
- `canvasStore.ts`: `addElement()`에서 캔버스 크기 기반 중앙 좌표 자동 계산
  - `centerX = (canvasWidth - elementWidth) / 2`
  - `centerY = (canvasHeight - elementHeight) / 2`
- `LeftPanel.tsx`: 모든 요소/위젯에서 하드코딩된 x/y 제거 → store가 자동 중앙 배치

### 9. 플레이리스트에서 캔버스 항목 0개/시간 없음 (V5.1.1)

**문제**: 플레이리스트에 캔버스 콘텐츠가 있으면 항목 통계에 안 나오고, 시간도 0초
**수정**:
- `PlaylistEditorPage.tsx`: 항목 통계 타입 목록에 `CANVAS` 추가, 아이콘/색상 추가
- `playlistController.js`: 캔버스 콘텐츠 추가 시 canvasJson에서 페이지 duration 합산하여 자동 설정

### 10. 플레이어에서 캔버스 콘텐츠 렌더링 (V5.1.1)

**문제**: 플레이어가 CANVAS 타입을 인식 못하고 이름만 표시
**수정**:
- `player/types/index.ts`: `ContentItem.type`에 `CANVAS` 추가, `CanvasData`/`CanvasPage`/`CanvasElement` 타입 추가
- `player/components/CanvasRenderer.tsx` 신규 생성:
  - 캔버스 JSON → HTML 렌더링 (텍스트, 도형, 이미지, 위젯)
  - 페이지 자동 전환 (각 페이지 duration만큼 표시)
  - 캔버스 크기를 화면에 맞게 스케일링
  - 시계 위젯 실시간 렌더링
  - SVG 기반 도형 렌더링 (삼각형, 화살표)
  - 애니메이션 CSS 지원
- `player/components/ContentRenderer.tsx`: CANVAS 분기 추가 → `CanvasRenderer` 호출
- `backend/playerController.js`: 스케줄 전송 시 CANVAS 콘텐츠에 `canvasData` (파싱된 JSON) 포함

## 🔧 운영 흐름 정리

### 장치 등록 → 매장 배정 플로우

```
[방법 1: 등록코드 방식 (권장)]
관리자 → 매장관리 → 등록코드 생성 (매장 지정)
  → 현장 설치자에게 6자리 코드 전달
  → 플레이어가 코드 입력
  → 자동으로 해당 매장+테넌트에 배정

[방법 2: URL 직접 등록]
플레이어 → /register 엔드포인트 직접 호출
  → default-tenant에 등록 (매장 미배정)
  → 관리자가 매장관리 → 장치 배정에서 수동 배정
  또는 장치 상세 → 편집 → 매장 선택

[방법 3: 관리자 직접 등록]
관리자 → 장치관리 → 장치 등록
  → 장치명, 장치ID, 매장, 그룹 등 입력
  → 바로 매장 배정 완료
```

### 역할별 권한 범위

| 역할 | 장치 조회 | 장치 등록 | 매장 관리 | 긴급 메시지 |
|------|----------|----------|----------|------------|
| SUPER_ADMIN | 전체 | ✅ | 전체 | 전체 |
| TENANT_ADMIN | 업체 전체 | ✅ | 업체 전체 | 업체 전체 |
| STORE_MANAGER | **자기 매장만** | ✅ | 자기 매장만 | 자기 매장 |
| USER | 읽기만 | ❌ | 읽기만 | ❌ |
| VIEWER | 읽기만 | ❌ | 읽기만 | ❌ |

## 🔍 추가 검토 필요 항목 (향후)

### P1 (높은 우선순위)
1. **URL 직접등록 tenant 문제**: `/register`로 등록 시 'default-tenant' 하드코딩 → 토큰 방식 유도 또는 tenant 자동 감지
2. **채널(Channel) 기능 연결**: 현재 채널과 스케줄/디바이스 간 연결 미구현
3. **콘텐츠 배포 상태 모니터링**: 배포 후 실제 디바이스 수신 확인 UI

### P2 (중간 우선순위)
4. **대시보드 매장별 통계**: STORE_MANAGER 대시보드에 자기 매장 통계만 표시
5. **캔버스 이미지 요소**: FabricImage 로딩 구현 (현재 placeholder Rect)
6. **캔버스 템플릿 만들기**: 템플릿 탭에서 빈 캔버스 템플릿 생성 UI
7. **장치 그룹 관리 페이지**: 그룹 CRUD 전용 페이지

### P3 (낮은 우선순위)
8. **플레이어 하트비트**: 주기적 상태 보고로 ONLINE/OFFLINE 자동 전환
9. **감사 로그 UI**: 관리자 활동 이력 조회 페이지
10. **스프레드시트 위젯 실제 연동**: 현재 iframe embed, API 연동 추가 가능
