## 2. Phase V5-1: 매장-장치 연동 시스템 정비

### 2.1 현재 문제점

| 문제 | 상세 | 심각도 |
|------|------|--------|
| 장치 목록에 매장 정보 미표시 | DevicesPage에서 어느 매장 소속인지 보이지 않음 | HIGH |
| 장치의 매장 할당 불가 | 장치 생성/수정 시 매장 선택 필드 없음 | CRITICAL |
| 매장 관리 페이지에서 장치 관리 불가 | 매장 상세에서 장치 목록만 보임, 추가/제거 불가 | CRITICAL |
| 매장관리자가 전체 장치 조회 가능 | storeFilter 미들웨어가 장치 라우트에 미적용 | CRITICAL |
| 장치 필터에 매장 필터 없음 | 장치 목록에서 매장별 필터링 불가 | MEDIUM |

### 2.2 개선 계획

#### 2.2.1 백엔드 수정

**A. 장치 컨트롤러 매장 필터링 추가** (`deviceController.js`)
```
기존: where = { tenantId } (테넌트 전체)
변경: where = { tenantId, ...(storeId ? { storeId } : {}) }
```
- `req.query.storeId` 쿼리 파라미터 지원
- STORE_MANAGER는 자동으로 자기 매장만 조회

**B. storeFilter 미들웨어 장치 라우트에 적용** (`routes/devices.js`)
```
기존: router.use(authenticate, tenantContext)
변경: router.use(authenticate, tenantContext, storeFilter)
```

**C. 장치 생성/수정 시 storeId 처리 추가**
- `registerDevice` / `updateDevice`에서 `req.body.storeId` 처리
- STORE_MANAGER는 자기 매장 storeId 자동 할당

**D. 매장 컨트롤러에 장치 관리 API 추가** (`storeController.js`)
- `POST /api/stores/:storeId/devices` — 장치를 매장에 할당
- `DELETE /api/stores/:storeId/devices/:deviceId` — 장치를 매장에서 해제
- `GET /api/stores/:storeId/devices` — 매장 소속 장치 목록

#### 2.2.2 프론트엔드 수정

**A. DevicesPage.tsx 개선**
- 테이블에 "매장" 컬럼 추가 (매장명 표시, 미배정 시 "미배정" 배지)
- 필터 영역에 매장 선택 드롭다운 추가
- 장치 추가/수정 모달에 매장 선택 필드 추가
- 장치 카드에 매장 정보 뱃지 표시

**B. StoresPage.tsx 개선**
- 매장 상세 패널에 "소속 장치" 탭 추가
- 장치 할당/해제 기능 (기존 장치 목록에서 선택)
- 드래그 앤 드롭으로 장치 매장 이동

**C. 매장관리자 전용 뷰**
- STORE_MANAGER 로그인 시 자기 매장의 장치만 표시
- 다른 매장 장치 접근 차단 (API + UI 모두)

| 문제 | 상세 | 심각도 |

### 3.2 개선 계획

#### 3.2.1 백엔드 (수정 적음)
- 컨트롤러 로직 자체는 정상 → 변경 불필요
- Socket.IO 브로드캐스트 로직 정상

#### 3.2.2 프론트엔드 — EmergencyPage.tsx 전면 개선

**A. 전송 대상 3단계 선택 UI**
```
전송 대상:
 ○ 전체 디바이스 (모든 매장, 모든 장치)
 ○ 특정 매장
     └─ [✓] 강남점  [✓] 홍대점  [ ] 합정점
 ○ 특정 장치
     └─ [매장 선택 ▼] → [✓] 1층TV  [✓] 2층TV
```

- **ALL**: 기존과 동일
- **STORE**: 매장 목록 체크박스 (stores API 호출)
- **DEVICE**: 매장 선택 → 해당 매장 장치 목록 체크박스

**B. targetIds 폼 연동**
- 체크한 매장/장치 ID를 `targetIds` 배열로 수집
- JSON.stringify 후 API 전송

**C. 전송 내역 상세 표시**
- 목록에서 각 메시지의 타겟 정보 표시
  - "전체" / "강남점, 홍대점" / "DEV-001, DEV-002"
- 전송 성공/실패 건수 표시

#### 3.2.3 플레이어 — 소켓 연동 수정

**A. playerStore.ts에 socket 상태 추가**
```typescript
interface PlayerStore {
  // ... 기존 ...
  socket: Socket | null
  setSocket: (socket: Socket | null) => void
}
```

**B. useSocket.ts에서 store 연동**
```typescript
const { setSocket } = usePlayerStore()
// 소켓 연결 후:
setSocket(socketRef.current)
// 소켓 해제 시:
setSocket(null)
```

**C. EmergencyOverlay.tsx 정상 동작 확인**
- socket이 정상적으로 주입되면 기존 코드로 동작 가능
- 추가 테스트: 긴급 메시지 전송 → 플레이어 표시 확인

### 4.2 수정 계획

#### B1. 위젯 줌 이중 스케일링 수정
```typescript
// 기존 (CanvasStage.tsx)
style={{
  left: element.x * zoom,
  width: element.width * zoom,
  transform: `rotate(...) scale(${zoom})`,  // ← 이중 적용!
}}

// 수정: scale 제거, 위치/크기에만 zoom 적용
style={{
  left: element.x * zoom,
  width: element.width * zoom,
  height: element.height * zoom,
  transform: `rotate(${element.rotation || 0}deg)`,
}}
```

#### B2. 시계 위젯 날짜 포맷 통일
- LeftPanel.tsx: `dateFormat: 'yyyy-MM-dd'` (date-fns 형식)
- RightPanel.tsx 옵션: `'yyyy-MM-dd'`, `'MM/dd/yyyy'`, `'yyyy년 MM월 dd일'`
- ClockWidget.tsx: 이미 date-fns 형식 사용 → 변경 불필요

#### B3. 날씨 위젯 키 통일
- WeatherWidget.tsx에서 `config.unit` → `config.units`로 통일
- 또는 LeftPanel/RightPanel의 `units` → `unit`으로 통일

#### B4. 화살표 도형 개선
- `Line` 오브젝트 대신 **SVG Path 기반 화살표** 구현
- Fabric.js `Path` 또는 `Polygon`으로 화살표머리 추가
- 새로운 `createArrowObject()` 함수:
```typescript
function createArrowObject(element) {
  const headLen = 15
  const path = `M 0 ${h/2} L ${w-headLen} ${h/2}
    M ${w-headLen} ${h/2-headLen/2}
    L ${w} ${h/2}
    L ${w-headLen} ${h/2+headLen/2}`
  return new Path(path, {
    ...baseProps,
    stroke: element.stroke || '#FFFFFF',
    strokeWidth: element.strokeWidth || 2,
    fill: '',
  })
}
```
- 추가 도형: 둥근사각형, 별, 다각형, 말풍선

#### B5. 위젯 드래그/리사이즈 구현
- **방법 A**: 위젯 선택 시 Fabric.js의 투명 Rect를 해당 위치에 생성 → 드래그/리사이즈 이벤트를 위젯 좌표에 동기화 (Proxy 패턴)
- **방법 B**: React DnD 또는 `onMouseDown`/`onMouseMove` 커스텀 핸들러
- **추천: 방법 A** — Fabric.js의 기존 조작 UI(핸들, 회전 등)를 그대로 활용 가능

```
위젯 선택 시:
1. 해당 위치에 투명 Fabric.Rect 생성 (opacity: 0, borderColor: blue)
2. object:modified 이벤트 → 위젯 element의 x/y/width/height 업데이트
3. 위젯 선택 해제 시 투명 Rect 삭제
```

#### B6. RSS 위젯 애니메이션 수정
```typescript
// 기존
const animRef = { current: requestAnimationFrame(frame) }
// 수정: useRef 사용
const animRef = useRef<number>(0)
animRef.current = requestAnimationFrame(frame)
return () => cancelAnimationFrame(animRef.current)
```

#### B7+B8. 템플릿 UX 개선
- 사용 성공 시 toast + 캔버스 데이터 리로드
- 저장 시 Fabric.js `canvas.toDataURL()` 으로 썸네일 생성
